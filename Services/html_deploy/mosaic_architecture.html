Overview
💡 Mosaic is an internal HTML report sharing platform — a single hub where any Moloco team member can publish interactive HTML reports (Plotly notebooks, Searchlight analyses, launch checklists) and share them with the team via permanent URLs, with no infrastructure knowledge required.
Problem it solves
  • No central home for analytics outputs
  • Sharing interactive HTML is friction-heavy
  • Reports get lost in Slack/Drive/email
Core capabilities
  • Upload & serve .html reports
  • Organize by category / sub-category
  • Private sub-category workspaces
  • Permanent shareable URLs
Access control
  • @moloco.com Google OAuth only
  • Edit/delete restricted to uploader
  • Private sub-cats: owner-only visibility
  • All enforced at API layer

Key design choices
GCS as metadata store
Replaces Firestore (permission issues). meta/categories/{id}.json and meta/reports/{id}.json act as a lightweight KV store. No extra infra needed — Compute SA already has editor on gds-apac.
Single Cloud Run service
Backend serves the compiled React SPA as static files. No separate frontend hosting needed.
GDrive proxy
/api/serve/{id} fetches GDrive HTML server-side, bypassing iframe-blocking. Result is cached in GCS cache/.
APP_BASE_URL env var
Cloud Run's internal proxy strips HTTPS. OAuth redirect URI must be set explicitly via env var, not derived from request.base_url.
GCS bucket layout
meta/categories/{id}.json
meta/reports/{id}.json
uploaded/{id}.html
cache/{id}.html

Backend Architecture
Routes layer (routes/)
categories.py — CRUD for categories. list_categories filters private categories the requester doesn't own. DELETE cascades to empty sub-categories.
reports.py — Upload (multipart/form-data), metadata update (PUT), file replacement (POST /{id}/replace), delete. All mutating endpoints require uploader == session.email.
serve.py — Proxies HTML to the iframe. Checks privacy before serving. For GDrive sources: fetches HTML via Drive API and caches in GCS. For uploads: reads directly from GCS.
Services layer (services/)
meta.py — GCS JSON KV store. Functions: list_categories(), get/put/delete_category(), list_sub_categories(parent_id), category_has_reports(id) (checks sub-cats too), plus report equivalents.
gcs.py — Low-level GCS helpers: write/read/delete_upload(), write/read/delete/exists_cache(). Single _bucket() factory keeps client lazily initialized.
gdrive.py — fetch_html(file_id) via Drive API v3, using the Compute SA credentials automatically injected by Cloud Run.

State & navigation
Filter state in URL
useSearchParams writes ?cat=<id>&sub=<id> on every filter change (replace, not push). On mount, reads initial state from URL params — so back-navigation from the viewer restores the exact filter.
Back navigation
ReportCard passes state.from = window.location.pathname + search when navigating to viewer. Viewer's ← Back button returns to state.from ?? '/'.
API wrapper
api.ts exposes get/post/put/del — always sends credentials: 'include' for session cookie. Throws with detail from error JSON.

Authentication Flow
1
Login redirect
Browser hits GET /auth/login. Backend builds Google OAuth URL with hd=moloco.com (hosted domain hint) and scope=openid email profile. Redirects to Google's consent screen.
2
OAuth callback
GET /auth/callback?code=… — backend exchanges code for tokens at oauth2.googleapis.com/token, then fetches userinfo from googleapis.com/oauth2/v3/userinfo.
3
Domain check
Validates email.endswith("@moloco.com"). Non-moloco accounts receive HTTP 403. This check happens before any session is issued.
4
Session cookie
Signs {email, name, picture} with itsdangerous.URLSafeTimedSerializer using SESSION_SECRET. Sets as HttpOnly; Secure; SameSite=Lax cookie with 7-day TTL. Redirects to /.
5
Session validation on every request
Every /api/* and /auth/me route calls require_session(request). Verifies cookie signature + expiry. Returns 401 on failure. No database lookup needed — session is self-contained.
⚠️ APP_BASE_URL must be set as a Cloud Run env var. Cloud Run's ingress proxy rewrites the scheme to http://, causing OAuth redirect URI mismatches. The env var overrides request.base_url in _redirect_uri().

Storage Model
✅ Firestore was the original design but was blocked by IAM permissions. GCS JSON blobs serve as a lightweight KV store — no schema migrations, no extra services, and the Compute SA already has editor access on gds-apac.

Performance note

list_categories() and list_reports() each do one GCS list_blobs call followed by parallel blob reads. category_has_reports(id) fetches all categories (to find sub-cat IDs) then all report blobs. This is O(n) but acceptable at current scale — an internal team tool with <1K reports. At scale, a Firestore migration would reduce this to indexed queries.

Privacy Model
💡 Privacy is scoped to sub-categories only. Top-level categories are always visible to all Moloco users. A private sub-category and all its reports are invisible to everyone except the creator — enforced at the API layer, not just the UI.
👁️
Visibility filtering — GET /api/categories
Backend filters: [c for c in meta.list_categories() if not c.is_private or c.created_by == session.email]. Non-owners never receive private category objects — the tab doesn't even exist in their UI.
📋
Report list filtering — GET /api/reports
Loads all categories into a dict, then filters out reports whose category_id maps to a private category not owned by the requester. One GCS read for all categories, then applied as a filter.
🔗
Direct URL access — GET /api/reports/{id} + GET /api/serve/{id}
Both endpoints load the report's category and check privacy before responding. Non-owners receive HTTP 404 (not 403) to avoid leaking existence of private reports.
✏️
Edit / delete guards
On top of privacy, mutating endpoints also check report.uploader == session.email. A non-owner cannot edit or delete even a public report. HTTP 403 for ownership violations.

API Reference
Auth — /auth/*
MethodPathDescription
GET/auth/loginRedirect to Google OAuth consent screen (hd=moloco.com)
GET/auth/callbackOAuth code exchange → session cookie → redirect to /
GET/auth/meReturns current session {email, name, picture} or 401
POST/auth/logoutDeletes session cookie
Categories — /api/categories
MethodPathAuthDescription
GET/api/categoriessessionList all visible categories (private filtered by ownership)
POST/api/categoriessessionCreate category. Body: CategoryCreate. Returns created object.
PUT/api/categories/{id}sessionUpdate name/color/is_private. None fields are skipped.
DELETE/api/categories/{id}sessionCascade-deletes empty sub-categories. 409 if any reports exist.
Reports — /api/reports
MethodPathAuthDescription
GET/api/reportssessionList reports. Params: ?category=id&q=text. Private category reports filtered out for non-owners.
POST/api/reports/uploadsessionMultipart upload of .html file. Fields: title, description, category_id, file. Max 20 MB.
GET/api/reports/{id}sessionGet report metadata. 404 if private and non-owner.
PUT/api/reports/{id}uploaderUpdate title/description/category_id. 403 for non-uploader.
POST/api/reports/{id}/replaceuploaderReplace HTML file (upload-type only). Invalidates GCS cache. 403 for non-uploader.
DELETE/api/reports/{id}uploaderDelete report + GCS objects. 403 for non-uploader.
Serve — /api/serve
MethodPathAuthDescription
GET /api/serve/{id} session Returns raw HTML. For gdrive: fetches from Drive API, caches in GCS. For upload: reads from uploaded/{id}.html. Response headers: X-Frame-Options: SAMEORIGIN, CSP: sandbox allow-scripts allow-same-origin. 404 for private category non-owners.

Dockerfile — multi-stage build
# Stage 1: build React SPA
FROM node:20-slim AS frontend-builder
WORKDIR /app/frontend
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ ./
RUN npm run build   # → ../backend/static/

# Stage 2: Python backend + compiled SPA
FROM python:3.12-slim
WORKDIR /app
COPY backend/requirements.txt ./
RUN pip install -r requirements.txt
COPY backend/ ./backend/
COPY --from=frontend-builder /app/backend/static ./backend/static/
CMD ["uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8080"]
