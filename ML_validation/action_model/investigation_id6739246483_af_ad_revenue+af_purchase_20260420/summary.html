  • Provided campaign IV7TA5O07K4JNsjn is long-term paused (inactive since 2025-12-18, 4+ months). It has no impressions in the last 3 days and generates no bid-log eval data. Action model IS configured (TYPE_ACTION/TYPE_SEC_KPI) but the campaign is not serving.
  • Active campaign SqHUbqoeXYnCj6Ow was just reactivated today (2026-04-20 04:31 UTC) after a ~6-day pause. No Moloco bid traffic has accumulated since reactivation, so no bid-log eval exists. Both IFA and LAT models are serving with sub-optimal normalizers: IFA uses organic MMP postback data (POSTBACK normalizer), LAT has no eval at all (FALLBACK normalizer = 0.804).
  • af_purchase pb eval quality is critically sparse: only 21/12/4/0/0 positives at D1/D3/D7/D14/D28 out of 3,551 examples. The reported AUC (0.98-0.99) is trivially inflated with <1% positive rate — the normalizer is unreliable for all horizons beyond D1.

Recommendation

  • IV7TA5O07K4JNsjn: Confirm with client whether this campaign should be reactivated or archived. If archiving, this cleanly removes it from the ML pipeline's consideration.
  • SqHUbqoeXYnCj6Ow: Allow ~1–2 weeks of active serving. Once bid-log eval accumulates, IFA will graduate POSTBACK → BIDLOG; LAT will get real eval data. Re-run this investigation at that time to confirm graduation.
  • af_purchase as secondary KPI: Flag to AE. With only 4 D7 positives out of 3,551 IFA examples, the model has almost no signal for this event. If af_ad_revenue is the true optimization target (it is — OPTIMIZE_ROAS campaign), af_purchase secondary KPI adds noise without actionable calibration. Consider removing it or discussing with the client whether purchase matters for bidding.

Note: Action Model Is Wired Up Correctly

The model type is correctly applied — TYPE_ACTION for af_ad_revenue (primary), TYPE_SEC_KPI for af_purchase (secondary), TYPE_ACTION_LAT for LAT impressions. The structural wiring is correct. The issue is normalizer quality, not model selection.
